## 🚀 Возможности

- **Управление задачами** - создание, редактирование, удаление задач
- **Статусы задач** - Бэклог, Надо, Делать, Проверка, Сделано!
- **Дедлайны** - установка и отслеживание сроков выполнения задач
- **История изменений** - полная история всех изменений каждой задачи
- **Управление разработчиками** - добавление, редактирование, назначение на задачи
- **Визуальные эффекты** - анимация частиц при приближении дедлайна
- **Поиск** - быстрый поиск задач по названию или описанию
- **Сохранение/Загрузка** - сохранение доски в JSON файл
- **Горячие клавиши** - полная поддержка клавиатурных сокращений
- **Современный интерфейс** - минималистичный дизайн с градиентами
- **Статистика** - подробная статистика по задачам и разработчикам

## 💻 Системные требования

### Минимальные требования

- **ОС**: Windows 10/11, macOS 10.15+, Ubuntu 20.04+ (или любой современный Linux)
- **RAM**: 512 МБ
- **Место на диске**: 100 МБ
- **Разрешение экрана**: 1280x720 или выше

### Требования для сборки

- **CMake**: версия 3.16 или выше
- **Компилятор**:
  - Windows: MSVC 2019+ или MinGW-w64 (GCC 9+)
  - macOS: Xcode Command Line Tools (Apple Clang)
  - Linux: GCC 9+ или Clang 10+
- **Qt**: версия 5.15+ или 6.x
- **Google Test**: для запуска тестов (опционально)

## 📥 Установка

### Windows 10 / 11

#### Шаг 1: Установка Qt

1. **Скачайте Qt Online Installer**
   - Перейдите на [https://www.qt.io/download-qt-installer](https://www.qt.io/download-qt-installer)
   - Скачайте "Qt Online Installer for Windows"
   - Запустите установщик

2. **Выберите компоненты для установки**
   - Создайте бесплатный аккаунт Qt или войдите
   - В Custom Installation выберите:
     - ✅ Qt 6.x (последняя версия)
       - ✅ MSVC 2019 64-bit или MinGW 64-bit
       - ✅ Qt Creator (IDE)
     - ✅ Developer and Designer Tools
       - ✅ CMake
       - ✅ Ninja

3. **Завершите установку**
   - Путь установки по умолчанию: `C:\Qt`
   - Установка займёт около 15-30 минут

#### Шаг 2: Установка Visual Studio (для MSVC)

Если вы выбрали MSVC:

1. Скачайте Visual Studio Community (бесплатно): [https://visualstudio.microsoft.com/downloads/](https://visualstudio.microsoft.com/downloads/)
2. При установке выберите:
   - ✅ Desktop development with C++
3. Дождитесь установки (может занять 30-60 минут)

#### Шаг 3: Установка Google Test (опционально, для тестов)

1. Откройте PowerShell от имени администратора
2. Установите vcpkg (менеджер пакетов):
```powershell
cd C:\
git clone https://github.com/Microsoft/vcpkg.git
cd vcpkg
.\bootstrap-vcpkg.bat
.\vcpkg integrate install
```

3. Установите Google Test:
```powershell
.\vcpkg install gtest:x64-windows
```

#### Шаг 4: Добавление Qt в PATH

1. Нажмите `Win + R`, введите `sysdm.cpl` и нажмите Enter
2. Перейдите на вкладку "Дополнительно" → "Переменные среды"
3. В "Системные переменные" найдите `Path` и нажмите "Изменить"
4. Добавьте путь к Qt bin (например: `C:\Qt\6.x.x\msvc2019_64\bin`)
5. Добавьте путь к CMake (если Qt установил его: `C:\Qt\Tools\CMake_64\bin`)
6. Нажмите "ОК" и перезапустите терминал

### macOS

#### Шаг 1: Установка Homebrew

Откройте Terminal (Cmd + Space, введите "Terminal") и выполните:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

#### Шаг 2: Установка необходимых инструментов

```bash
# Установка Xcode Command Line Tools (если ещё не установлено)
xcode-select --install

# Установка CMake
brew install cmake

# Установка Qt6
brew install qt@6

# Установка Google Test (для тестов)
brew install googletest
```

#### Шаг 3: Добавление Qt в PATH

Добавьте в файл `~/.zshrc` (или `~/.bash_profile` если используете bash):

```bash
echo 'export PATH="/opt/homebrew/opt/qt@6/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### Linux

#### Ubuntu / Debian

```bash
# Обновление списка пакетов
sudo apt update

# Установка необходимых инструментов
sudo apt install -y build-essential cmake git

# Установка Qt6
sudo apt install -y qt6-base-dev qt6-tools-dev qt6-tools-dev-tools

# Установка Google Test (для тестов)
sudo apt install -y libgtest-dev

# Сборка Google Test (необходимо для Ubuntu/Debian)
cd /usr/src/gtest
sudo cmake CMakeLists.txt
sudo make
sudo cp lib/*.a /usr/lib
```

#### Fedora

```bash
sudo dnf install cmake gcc-c++ qt6-qtbase-devel gtest-devel
```

#### Arch Linux

```bash
sudo pacman -S cmake gcc qt6-base gtest
```

## 🔨 Сборка проекта

### Сборка на Windows

#### Способ 1: Visual Studio (рекомендуется)

1. **Откройте Qt Creator**
   - Запустите Qt Creator из меню Пуск
   - File → Open File or Project
   - Выберите `CMakeLists.txt` в корне проекта

2. **Настройте проект**
   - Выберите Kit (например, "Desktop Qt 6.x.x MSVC2019 64bit")
   - Нажмите "Configure Project"

3. **Соберите проект**
   - Нажмите Ctrl+B или меню Build → Build Project
   - Дождитесь завершения сборки

4. **Запустите приложение**
   - Нажмите Ctrl+R или меню Build → Run

#### Способ 2: Командная строка

Откройте "Developer Command Prompt for VS 2019" из меню Пуск:

```cmd
cd путь\к\scrum_board
cmake -S . -B build -G "Visual Studio 16 2019" -A x64
cmake --build build --config Release
```

Или с MinGW:

```cmd
cd путь\к\scrum_board
cmake -S . -B build -G "MinGW Makefiles"
cmake --build build
```

### Сборка на macOS

```bash
cd /путь/к/scrum_board

# Создание директории для сборки
mkdir build
cd build

# Конфигурация проекта
cmake ..

# Сборка (используя все ядра процессора)
cmake --build . -j$(sysctl -n hw.ncpu)
```

### Сборка на Linux

```bash
cd /путь/к/scrum_board

# Создание директории для сборки
mkdir build
cd build

# Конфигурация проекта
cmake ..

# Сборка (используя все ядра процессора)
cmake --build . -j$(nproc)
```

### Сборка без тестов

Если у вас не установлен Google Test или вы не хотите собирать тесты:

```bash
cmake -S . -B build -DBUILD_TESTS=OFF
cmake --build build
```

## 📖 Описание работы

### Стартовый экран

При запуске приложения вы увидите стартовый экран с тремя кнопками:

1. **Новая доска (Ctrl+N)** - создать новую пустую доску
2. **Обзор... (Ctrl+O)** - загрузить сохранённую доску из файла
3. **Выйти (Ctrl+Q)** - закрыть приложение

### Главное окно

После создания или загрузки доски откроется главное окно с:

#### Меню "Доска"
- **Новая доска (Ctrl+N)** - очистить доску и начать заново
- **Сохранить (Ctrl+S)** - сохранить доску в JSON файл
- **Загрузить (Ctrl+O)** - загрузить доску из файла
- **Статистика (Ctrl+I)** - показать подробную статистику
- **Выход (Ctrl+Q)** - вернуться на стартовый экран

#### Меню "Разработчики"
- **Добавить разработчика (Ctrl+D)** - добавить нового разработчика в команду
- **Управление разработчиками (Ctrl+M)** - просмотр, редактирование, удаление разработчиков

#### Меню "Задачи"
- **Добавить задачу (Ctrl+T)** - создать новую задачу

#### Панель поиска
- Поле поиска в верхней части экрана позволяет фильтровать задачи по названию или описанию

### Колонки задач

Задачи отображаются в 5 колонках в зависимости от статуса:

1. **БЭКЛОГ** - задачи в очереди
2. **НАДО** - задачи, назначенные на разработчиков
3. **ДЕЛАТЬ** - задачи в работе
4. **ПРОВЕРКА** - задачи на ревью
5. **СДЕЛАНО!** - завершённые задачи

### Работа с задачами

#### Создание задачи
1. Нажмите "Добавить задачу" или Ctrl+T
2. Введите название (обязательно)
3. Введите описание (опционально)
4. Выберите разработчика из списка (если есть)
5. Отметьте "Установить дедлайн" если нужен срок
6. Выберите дату дедлайна в календаре
7. Нажмите "OK"

#### Редактирование задачи
1. Нажмите "✏️" на карточке задачи
2. Измените нужные поля
3. Нажмите "OK"

#### Удаление задачи
1. Нажмите "🗑️" на карточке задачи
2. Подтвердите удаление

#### Изменение статуса
1. Нажмите кнопку со статусом на карточке (например, "→ ДЕЛАТЬ")
2. Статус автоматически изменится на следующий

#### Перетаскивание задач
- Зажмите карточку задачи левой кнопкой мыши
- Перетащите в нужную колонку
- Отпустите кнопку - статус изменится автоматически

### Работа с разработчиками

#### Добавление разработчика
1. Меню "Разработчики" → "Добавить разработчика" (Ctrl+D)
2. Введите имя
3. Введите должность
4. Нажмите "OK"

#### Управление разработчиками
1. Меню "Разработчики" → "Управление разработчиками" (Ctrl+M)
2. Выберите разработчика из списка
3. Нажмите "Редактировать" для изменения или "Удалить" для удаления

**Важно**: При удалении разработчика все его задачи будут автоматически освобождены.

### Дедлайны и визуальные эффекты

Задачи с дедлайнами имеют специальные индикаторы:

- **7+ дней до дедлайна** - обычная карточка
- **3-7 дней до дедлайна** - слабое свечение, редкие частицы
- **1-3 дня до дедлайна** - среднее свечение, умеренные частицы
- **< 1 дня до дедлайна** - яркое свечение, интенсивные частицы
- **Просрочено** - красный фон, активные частицы

### Сохранение и загрузка

#### Сохранение доски
1. Меню "Доска" → "Сохранить" (Ctrl+S)
2. Выберите место сохранения
3. Введите имя файла (будет добавлено расширение .json)
4. Нажмите "Сохранить"

Файл содержит:
- Все задачи с их свойствами
- Всех разработчиков
- Полную историю изменений каждой задачи
- Дедлайны

#### Загрузка доски
1. Меню "Доска" → "Загрузить" (Ctrl+O)
2. Выберите файл .json
3. Нажмите "Открыть"

**Внимание**: Текущая доска будет полностью заменена загруженной.

### Статистика

Нажмите "Статистика" (Ctrl+I) чтобы увидеть:

- Количество задач по статусам
- Количество задач по разработчикам
- Средний прогресс выполнения
- Количество просроченных задач
- Количество задач без назначенных разработчиков

## 🐛 Решение проблем

### Windows

**Проблема**: "Qt6Widgets.dll не найден"
- **Решение**: Добавьте путь к Qt bin в PATH (см. инструкцию установки)

**Проблема**: "VCRUNTIME140.dll не найден"
- **Решение**: Установите Visual C++ Redistributable: [https://aka.ms/vs/17/release/vc_redist.x64.exe](https://aka.ms/vs/17/release/vc_redist.x64.exe)

**Проблема**: CMake не находит Qt
- **Решение**: Установите переменную окружения `CMAKE_PREFIX_PATH`:
  ```cmd
  set CMAKE_PREFIX_PATH=C:\Qt\6.x.x\msvc2019_64
  ```

### macOS

**Проблема**: "скрам_board.app повреждён и не может быть открыт"
- **Решение**: В Терминале выполните:
  ```bash
  xattr -cr /путь/к/scrum_board.app
  ```

**Проблема**: Qt не найден при сборке
- **Решение**: Установите переменную окружения:
  ```bash
  export CMAKE_PREFIX_PATH=/opt/homebrew/opt/qt@6
  ```

### Linux

**Проблема**: "error while loading shared libraries: libQt6Widgets.so.6"
- **Решение**: Обновите кэш библиотек:
  ```bash
  sudo ldconfig
  ```

**Проблема**: Google Test не найден
- **Решение**: Соберите и установите вручную:
  ```bash
  cd /usr/src/gtest
  sudo cmake CMakeLists.txt
  sudo make
  sudo cp lib/*.a /usr/lib
  ```
